# openEagle 开发指南

## 1. 环境要求

- Node.js（建议使用 `corepack` + `pnpm@10.7.0`）
- Python `>=3.12`
- `uv`（用于后端依赖与构建）
- Rust / Cargo（Tauri 开发）

## 2. 本地启动

```powershell
corepack enable
corepack prepare pnpm@10.7.0 --activate
pnpm install
uv sync --project .\backend
pnpm tauri:dev
```

## 3. 编码与文件规范（强约束）

- 所有文件读写必须使用 `UTF-8`（无 BOM）。
- 在 Windows 下执行写文件前，先确认控制台编码：

```powershell
chcp 65001
```

- 禁止使用默认编码、GBK、ANSI。
- 如果发现中文乱码，先确认正确原文来源（历史版本/上下文）后再改，不能“猜测修复”。

## 4. 关键架构

### 4.1 进程与连接

- Tauri Rust 启动 Python sidecar。
- 后端启动后输出握手日志：`[AGENT_READY] WS_PORT: <port>`。
- 前端建立 `ws://127.0.0.1:<port>/ws`。

### 4.2 SOLO 流程

1. 前端发送 `client:start_solo`。
2. 后端截图、VL 决策、下发 `server:solo_step`。
3. 执行动作并通过 `server:trace`、`server:solo_status` 回传。
4. 遇到危险动作进入 `server:solo_confirmation_required`。

### 4.3 显示器选择与预览

- 前端可发送 `client:list_solo_displays` 获取显示器列表与实时预览。
- 设置中的 `solo.preferredDisplayIndex` 会通过 `client:update_settings` 同步到后端。
- 后续 SOLO 截图按该显示器执行。

## 5. 图片显示说明

为避免 `asset.localhost` 在部分机器上访问临时目录出现 `404`，项目增加了 Rust 命令 `read_image_data_url`：

- 前端优先将本地截图路径转换为 `data:image/...;base64` 再渲染。
- 该机制用于设置页显示器预览以及对话中的 SOLO 截图预览。

## 6. 常用检查命令

前端：

```powershell
pnpm -s tsc --noEmit
pnpm -s build
```

后端：

```powershell
backend\.venv\Scripts\python.exe -m compileall backend\app
```

Rust：

```powershell
cd .\src-tauri
cargo check
```

## 7. 提交流程建议

```powershell
git status
git add <files>
git commit -m "feat: <summary>"
```

建议提交前至少完成：
- 前端 `tsc` 与 `build`
- 后端 `compileall`
- Rust `cargo check`（若改了 `src-tauri`）
