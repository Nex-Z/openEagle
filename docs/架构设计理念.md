混合架构：

打包后端： 使用 Python 编写核心逻辑，通过 FastAPI + WebSockets 暴露接口。然后用 PyInstaller 将其打包成一个单文件的独立可执行程序（无依赖二进制文件）。

Tauri 侧静默拉起： 在 Tauri 的 Rust 层，将这个 Python 程序配置为 Sidecar。当 Tauri 客户端启动时，由 Rust 负责拉起这个 Python 进程。

动态端口握手（极其关键）：

Python 后端启动时，不写死端口，而是向系统请求一个随机空闲端口（如 port = 0，OS 会分配一个）。

Python 后端启动成功后，向控制台（Stdout）打印一行特定的握手日志，例如：[AGENT_READY] WS_PORT: 54321。

Tauri 的 Rust 层监听子进程的 Stdout，捕获到这行日志后，将端口号传递给前端 JS。

前端 JS 拿到端口，立刻发起 new WebSocket('ws://127.0.0.1:54321') 连接。

生命周期接管： 当用户关闭 Tauri 窗口时，Rust 捕获退出信号，直接通过系统层级 kill 掉关联的 Python Sidecar 进程。
