# 架构设计理念

## 1. 混合架构

- 后端使用 Python（FastAPI + WebSocket）承载 Agent 推理、SOLO 编排与业务协议。
- 桌面壳使用 Tauri（Rust）负责进程托管与本地系统能力。
- 前端使用 React + TypeScript 负责对话界面、设置中心、状态可视化。

## 2. 后端部署模型

- 开发环境：由 Tauri 启动本地 Python 环境运行 `app.main`。
- 生产环境：Python 后端可通过 `PyInstaller` 打包为 sidecar 可执行文件。

## 3. 动态端口握手

1. Python 后端以随机端口启动（`port=0`）。
2. 启动成功后输出握手日志：`[AGENT_READY] WS_PORT: <port>`。
3. Rust 监听 sidecar 输出，解析端口并发布给前端。
4. 前端连接 `ws://127.0.0.1:<port>/ws`。

这样可以避免端口冲突，提升多实例与本地环境兼容性。

## 4. 生命周期接管

- Tauri 统一管理 sidecar 启停。
- 主窗口退出时，Rust 主动终止后端子进程，避免孤儿进程。
- 连接断开时前端执行自动重连策略并回显状态。

## 5. SOLO 设计原则

- 模型只做“决策”，执行由本地工具层负责。
- 所有关键步骤通过 `server:solo_status / step / trace` 回传，保证可追踪。
- 危险动作进入显式确认流程，减少误操作风险。
- 自动暂停机制用于防止死循环（如连续重复动作）。

## 6. 显示器与截图策略

- 支持从设置页选择 SOLO 显示器（`preferredDisplayIndex`）。
- 后端按该设置执行截图与坐标映射。
- 提供实时显示器预览用于选择校验。
- 为规避 `asset.localhost` 在部分机器对临时目录的访问限制，前端支持通过 Rust 命令读取图片并渲染为 data URL。
